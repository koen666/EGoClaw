# EGoclaw App 结构说明

## 1. 产品形态

EGoclaw 不是一个 Web 页面里挂一个宠物组件。

EGoclaw 是一个桌面 App，包含两个部分：

1. 主应用窗口
2. 系统级桌宠灵宝

主应用窗口负责：

1. 展示收藏同步结果
2. 展示知识图谱和成长路径
3. 展示当前意图和下一步动作
4. 承接用户深度交互

系统级桌宠灵宝负责：

1. 常驻桌面
2. 检测上下文触发
3. 从抖音窗口边缘跳出
4. 主动提醒用户
5. 点击后拉起主应用窗口

核心形态：

1. 用户打开 EGoclaw 后，主应用可关闭，灵宝继续常驻
2. 用户打开抖音时，后台检测到前台应用为抖音
3. 如果当前存在可触发的提醒，灵宝从抖音边缘跳到桌面上
4. 用户点击灵宝，进入主应用中的对应页面

---

## 2. App 总体结构

EGoclaw 由三部分构成：

1. 主应用窗口
2. 灵宝桌宠窗口
3. 后台常驻服务

### 2.1 主应用窗口

主应用窗口是用户查看内容和执行任务的地方。

### 2.2 灵宝桌宠窗口

灵宝不是主应用里的一个浮层，而是独立窗口。

桌宠窗口特点：

1. 透明
2. 无边框
3. 常驻最前
4. 可隐藏可唤起
5. 可在桌面任意位置停留

### 2.3 后台常驻服务

后台服务负责：

1. 检测当前前台应用
2. 判断是否打开抖音
3. 获取抖音窗口位置
4. 计算是否满足提醒条件
5. 触发灵宝出现
6. 管理同步、蒸馏、图谱、意图分析

---

## 5. 灵宝桌宠窗口

灵宝不是页面，而是独立窗口。

### 5.1 桌宠待机状态

内容：

1. 小火人形态
2. 默认停留位置

功能：

1. 常驻桌面
2. 保持轻量存在感

### 5.2 桌宠触发状态

触发条件：

1. 后台检测到抖音被打开
2. 当前存在可触发意图
3. 当前时间和上下文满足提醒条件

触发行为：

1. 获取抖音窗口边界
2. 灵宝从抖音边缘跳出
3. 跑到桌面或用户视线区域
4. 弹出提醒气泡

### 5.3 桌宠提醒内容

典型内容：

1. 主人，你上周收藏的“面试自我介绍”我帮你整理好了，要不要今晚练 1 分钟版？
2. 你之前想学低脂早餐，明早 10 分钟就能做一个最简单版本，要不要我陪你试一次？
3. 你这几天又刷到 PPT 内容了，我帮你把收藏夹里相关内容串成了一条入门路径。

### 5.4 桌宠可执行操作

用户可以直接点击：

1. 现在开始
2. 晚点提醒
3. 不是这个
4. 打开主应用

对应结果：

1. 打开当前行动页
2. 延后提醒
3. 回写负反馈
4. 拉起主窗口

## 6. 典型流程

## 6.1 首次使用流程

1. 用户打开 App
2. 进入启动与连接页
3. 连接抖音账号
4. 同步收藏夹
5. 系统完成蒸馏和图谱初始化
6. 进入首页总览

## 6.2 日常后台运行流程

1. 用户启动过 App
2. 主应用可以关闭
3. 灵宝和后台服务继续常驻
4. 系统持续检测前台应用和触发条件

## 6.3 抖音触发流程

1. 用户打开抖音
2. 后台检测到抖音在前台
3. 系统判断当前是否有可触发提醒
4. 获取抖音窗口边界
5. 灵宝从抖音边缘跳出
6. 用户点击灵宝
7. 打开主应用中的当前行动页或成长地图页

## 6.4 深度查看流程

1. 用户点击灵宝
2. 打开首页总览或当前行动页
3. 用户进入成长地图页
4. 点击节点查看内容蒸馏结果
5. 继续进入当前行动页
